Design implementable software units

The following areas should be considered in the design of the implementable software units:
 
 

  • client interfaces
    • Cross-check system feature definitions to ensure that the client interfaces provide the appropriate functionality
    • Review client-centered design processes (e.g., client interviews, prototyping, etc.)
    • Evaluate incremental feedback regarding the usability of the system
    • Inform team members of work being performed
    • Conduct usability testing with clients and make changes to reflect client feedback 
    • Specify client interface technology base and execution environment
  • error handling procedures
  • data validation procedures
  • recovery and restart possibilities
  • performance
  • volume limitations/boundary conditions
  • software reuse (may include reasons for not reusing software)
  • database design
  • installation/transition procedures 
  • conversion routines
  • CIS, capacity, contingency and resource estimates
  • security
  • administration and operation
  • fraud
  • client impacts
  • software development architecture 
  • COE architecture
